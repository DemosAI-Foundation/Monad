SYSTEM ARCHITECTURE — v0.91 Continuous Generative
1. INPUT
Raw user text received.
2. ENCODE — Generative Free Energy
The LLM serves as the generative model. P(user_text|context, model) is computed via per-token logprobs to yield exact Perplexity (Surprise). Simultaneously, embeddings extract Semantic Velocity and VAD.
3. LATENT UPDATE — Precision-Weighted Bayesian
b_{t+1} ∝ p(x_{t+1}|b_t) × b_t. The system uses a Kalman gain mechanism: high observation surprise reduces prior precision, rapidly pulling the latent state toward the new observation.
4. OUTPUT — Continuous Interpolation
The 3D numerical coordinates are embedded directly into the prompt with defined semantic axes, forcing the LLM to interpolate smoothly in its latent space to dictate tone and pacing.
ACTIVE INFERENCE — Expected Free Energy
At silence, generates candidates. Scored via EFE = Pragmatic Value (distance to preferred state) minus Epistemic Value (information gain, measured by predicting user response perplexity).
